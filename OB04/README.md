# OB04 - Применение принципа Open/Closed

## Описание проекта

Простая игра, демонстрирующая применение принципа открытости/закрытости (Open/Closed Principle) из SOLID.

## Принцип Open/Closed

**Программные сущности должны быть открыты для расширения, но закрыты для модификации.**

В данном проекте:
- ✅ **Открыты для расширения**: Можно легко добавлять новые типы оружия, создав новый класс, наследующий `Weapon`
- ✅ **Закрыты для модификации**: Классы `Fighter` и `Monster` не нужно изменять при добавлении нового оружия

## Классы

### Weapon (Абстрактный)
- `attack()` - абстрактный метод атаки

### Конкретные типы оружия
- **Sword** - Меч
- **Bow** - Лук
- **Axe** - Топор
- **Spear** - Копье

### Fighter
- `change_weapon(weapon)` - сменить оружие
- `attack()` - атаковать текущим оружием

### Monster
- `take_damage(damage)` - получить урон

## Как добавить новое оружие

1. Создайте новый класс, наследующий `Weapon`
2. Реализуйте метод `attack()`
3. Используйте новое оружие в игре

**Не требуется изменять существующий код!**

### Пример:

```python
class MagicWand(Weapon):
    def attack(self) -> str:
        return "Боец использует магическую атаку!"
```
